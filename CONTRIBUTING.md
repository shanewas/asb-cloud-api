# Contributing

Thanks for helping improve ASB Cloud API.

## Ground Rules

- Keep changes focused and easy to review.
- Add or update tests for behavior changes.
- Do not commit secrets, `.env` files, generated bytecode, browser artifacts, or local tool state.
- For large changes, open an issue first and describe the intended design.
- Follow the release contract in [SPEC.md](SPEC.md).

## Local Setup

```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m playwright install chromium
```

## Checks

Run before opening a pull request:

```bash
python -m compileall -q asb_api tests
python -m unittest discover -s tests -v
```

If Clawpatch is available locally:

```bash
clawpatch map
clawpatch review --limit 10
clawpatch report
```

## Pull Request Checklist

- The change is scoped to one logical concern.
- Tests pass locally.
- Public API changes are reflected in `README.md` and `SPEC.md`.
- New configuration is documented in `.env.example`.
- Security-sensitive changes mention threat model and failure behavior.
